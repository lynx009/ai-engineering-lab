## 中文

### 欢迎贡献！

感谢您对 AI Engineering Lab 项目的关注！我们欢迎所有形式的贡献。

### 贡献方式

#### 1. 对于新贡献者

如果您是第一次为本项目贡献，请按照以下步骤：

1. **Fork 本仓库**
   - 点击页面右上角的 "Fork" 按钮
   - 在您的 GitHub 账户下创建项目副本

2. **克隆您的 Fork**
   ```bash
   git clone https://github.com/YOUR_USERNAME/ai-engineering-lab.git
   cd ai-engineering-lab
   ```

3. **创建功能分支**
   ```bash
   git checkout -b feature/your-feature-name
   ```

4. **进行更改并提交**
   ```bash
   git add .
   git commit -m "描述您的更改"
   ```

5. **推送到您的 Fork**
   ```bash
   git push origin feature/your-feature-name
   ```

6. **创建 Pull Request**
   - 访问原始仓库页面
   - 点击 "New Pull Request"
   - 选择您的分支并提交 PR

#### 2. 对于项目协作者

如果您是项目的协作者，您可以：

1. **直接克隆仓库**
   ```bash
   git clone https://github.com/lynx009/ai-engineering-lab.git
   ```

2. **创建功能分支**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **推送并创建 PR**
   ```bash
   git push origin feature/your-feature-name
   ```

### 分支保护规则

- **main 分支**: 受保护，仅仓库所有者 @lynx009 有直接编辑权限
- **所有更改**: 必须通过 Pull Request 进行
- **代码审查**: 所有 PR 将经过 AI 自动审查和人工审查

### Pull Request 指南

1. **清晰的标题**: 使用描述性的 PR 标题
2. **详细的描述**: 说明您的更改内容和原因
3. **关联 Issue**: 如果相关，请引用对应的 Issue
4. **测试**: 确保您的代码经过测试
5. **代码风格**: 遵循项目的代码规范

### AI 代码审查

本项目使用 GitHub Copilot 和 AI 工具自动进行代码审查：

- 自动检查代码质量
- 识别潜在的 bug 和安全问题
- 提供改进建议
- 确保代码符合最佳实践

### 代码规范

- 使用有意义的变量和函数名
- 添加必要的注释
- 保持代码简洁清晰
- 遵循语言特定的最佳实践

### Issue 提交

提交 Issue 时请：

1. 使用清晰的标题
2. 详细描述问题或建议
3. 如果是 bug，提供复现步骤
4. 提供相关的环境信息

### 行为准则

- 尊重所有贡献者
- 保持友好和专业
- 提供建设性的反馈
- 欢迎新手和学习者

## English

### Welcome Contributors!

Thank you for your interest in the AI Engineering Lab project! We welcome all forms of contributions.

### How to Contribute

#### 1. For New Contributors

If you're contributing to this project for the first time, please follow these steps:

1. **Fork the Repository**
   - Click the "Fork" button in the top right corner
   - Create a copy of the project under your GitHub account

2. **Clone Your Fork**
   ```bash
   git clone https://github.com/YOUR_USERNAME/ai-engineering-lab.git
   cd ai-engineering-lab
   ```

3. **Create a Feature Branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

4. **Make Changes and Commit**
   ```bash
   git add .
   git commit -m "Describe your changes"
   ```

5. **Push to Your Fork**
   ```bash
   git push origin feature/your-feature-name
   ```

6. **Create a Pull Request**
   - Visit the original repository page
   - Click "New Pull Request"
   - Select your branch and submit the PR

#### 2. For Project Collaborators

If you are a project collaborator, you can:

1. **Clone the Repository Directly**
   ```bash
   git clone https://github.com/lynx009/ai-engineering-lab.git
   ```

2. **Create a Feature Branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Push and Create PR**
   ```bash
   git push origin feature/your-feature-name
   ```

### Branch Protection Rules

- **main branch**: Protected, only repository owner @lynx009 has direct edit permissions
- **All changes**: Must go through Pull Requests
- **Code review**: All PRs will undergo AI automatic review and manual review

### Pull Request Guidelines

1. **Clear Title**: Use a descriptive PR title
2. **Detailed Description**: Explain what you changed and why
3. **Link Issues**: Reference related issues if applicable
4. **Testing**: Ensure your code is tested
5. **Code Style**: Follow the project's coding standards

### AI Code Review

This project uses GitHub Copilot and AI tools for automated code review:

- Automatically checks code quality
- Identifies potential bugs and security issues
- Provides improvement suggestions
- Ensures code follows best practices

### Code Standards

- Use meaningful variable and function names
- Add necessary comments
- Keep code clean and concise
- Follow language-specific best practices

### Issue Submission

When submitting an Issue:

1. Use a clear title
2. Describe the problem or suggestion in detail
3. If it's a bug, provide reproduction steps
4. Provide relevant environment information

### Code of Conduct

- Respect all contributors
- Stay friendly and professional
- Provide constructive feedback
- Welcome newcomers and learners
